# Localization (i18n)

Aegis ships in eight locales: **en, es, pt-BR, fr, de, ja, hi, bn**. English is the source language; every other catalog is a partial translation that falls back to the English default at the call site, so partial coverage is always safe.

## Where things live

- `src/lib/i18n.ts` — the `i18n` singleton (from `@lingui/core`), the `SUPPORTED_LOCALES` list, `getLocalePref` / `setLocalePref` / `applyLocale`, and the pre-hydration `LOCALE_INIT_SCRIPT` that sets `<html lang>` before React mounts.
- `src/locales/{code}/messages.ts` — one flat `Record<string, string>` per locale. All eight are statically imported at boot; switching is synchronous.
- `src/routes/__root.tsx` — mounts `<I18nProvider>`, boots the locale on load, and syncs `profiles.locale` from the server on sign-in.
- `profiles.locale` — nullable text column, `null` means "system" (browser detection). Constrained to the eight supported codes.

## Writing translatable strings

Two call styles, both driven by an explicit message id:

```tsx
import { useLingui } from "@lingui/react";

function ProfileRow() {
  const { i18n } = useLingui();
  const t = (id: string, fallback: string) => {
    const msg = i18n._(id);
    return msg === id ? fallback : msg;
  };
  return <span>{t("profile.signOut", "Sign out")}</span>;
}
```

Or with the `<Trans>` component for inline JSX:

```tsx
import { Trans } from "@lingui/react";
<Trans id="profile.signOut">Sign out</Trans>;
```

Rules:

1. Every id is namespaced with a dot-separated prefix (`profile.*`, `tabs.*`, `common.*`).
2. Every call site provides a **default English string** as the fallback (`t(id, fallback)` or `<Trans>Fallback</Trans>`). Never rely on the catalog being complete.
3. When you add a new id, add it to `src/locales/en/messages.ts` first, then to any other catalog you can translate. Untranslated locales just fall back — no build failure.

## Locale picker

Profile → **Language**. `chooseLocale()` in `src/routes/_authenticated/_tabs/profile.tsx`:

1. `setLocalePref(pref)` — persists to `localStorage` + activates on `i18n`.
2. Upserts `profiles.locale` (writes `null` for `"system"`).

On next sign-in from another device, `syncPrefsFromProfile()` in `__root.tsx` re-applies the saved locale.

## String freeze policy

Any PR that touches user-facing copy must:

1. Add or update the id in `src/locales/en/messages.ts`.
2. Update the sibling locale files that already have the ids in the same section (best-effort — if you don't speak the language, it's fine to leave them and rely on English fallback).
3. Mention "i18n" in the PR description so reviewers can spot missing ids.

A CI extraction check is planned for Phase 8.4 alongside axe-core.

## Non-goals (for now)

- Right-to-left support — none of the eight initial locales are RTL.
- Localizing the marketing home page and blog route (SEO English-only for now).
- ICU plural / select forms — the current API is plain-string only. Add them per-key when a use case appears.
